# 2. 性能指标
1. 系统的最大负载 (网络 / 连接 / 请求量 / 流量 / 数据磁盘量 / 在线人数等)
2. 某些场景的最长响应时长

# 3. 稳定指标
1. 可持续运行多久

# 4. 方案选型
1. 技术方案的收集 / 比较

# 5. 静态结构
1. 单元(进程 / 线程 / 子系统) 之间的结构关系

# 6. 概要流程
1. 静态结构中单元的概要 数据 交互

# 7. 数据定义
1. 文件路径及用途
2. 数据库表定义
3. 交互JSON
4. 程序内关键对象 (如 大的树 / 大的表)

# 8. 交互说明
1. 状态机: 一个小对象中的状态变换 (注意说明 触发事件源)
2. 时序图: 多个小模块中的交互逻辑
3. 流程图: 一个小模块中的执行逻辑

# 9. 出错处理
1. 异常场景出错场景的处理, 比如说对端拿不到地址
2. 处理一般方法是: 程序逻辑退化, 报错日志, 直接挂掉, 重试, 忽略

# 10. 可测试性
1. 如变更配置文件如何 reload; 如设置某些标记变量触发特殊逻辑
2. 如特殊后门执行逻辑(处理客户端数据的逻辑, 开放服务端直接处理文件模拟数据的后门)

# 11. 可调试性
1. 长久日志
2. 实时日志
3. 动态获取程序运行时的某些关键上下文
4. 程序内事件统计值

# 12. 安全性: 防泄密, 泄漏等

# 13. 可靠性
